# Godot 4 Game Base Architecture

## Agent Topology

| Agent | Role | Model | Tools |
|-------|------|-------|-------|
| orchestrator | Routes work, manages build phases, never writes game code | sonnet | Read |
| simulation | Python headless game loop, balance validation, scoring formula | sonnet | Read, Write, Bash |
| core-systems | GDScript Resource class definitions, card/room/party systems | sonnet | Read, Write |
| content | .tres data instances (cards, rooms, adventurers) — REQUIRES core-systems first | sonnet | Read, Write |
| ui-ux | Godot scenes (.tscn), HUD, menus, visual feedback — REQUIRES core-systems first | sonnet | Read, Write |
| infra | Makefile, CI/CD, project.godot config, headless runner setup | sonnet | Read, Write, Bash |
| testing | GUT test stubs, Python simulation tests, USER_TESTING.md | sonnet | Read, Write, Bash |
| reviewer | Quality gate — read-only | sonnet | Read, Glob, Grep, Bash |

## Build Order

**CRITICAL — enforce this dependency chain in ARCHITECTURE.md:**

```
1. simulation    → Python game loop first, no Godot needed
2. core-systems  → GDScript schemas/classes
3. content       → .tres data files (DEPENDS ON: core-systems schemas)
4. ui-ux         → .tscn scenes (DEPENDS ON: core-systems systems)
5. testing       → GUT + Python tests
6. infra         → CI/CD, Makefile, project.godot
7. reviewer      → validates all
```

The content agent CANNOT create .tres files before core-systems has defined the Resource class schemas. The ui-ux agent CANNOT wire scenes to systems that don't exist. The architect must enforce this dependency chain in ARCHITECTURE.md.

## File Types Reference

- `.gd` — GDScript (logic, class definitions). Opened/edited directly.
- `.tscn` — Scene files (node trees). Text format, human-readable.
- `.tres` — Resource data instances. Text format. Think of these as typed JSON.
- `.import` — Auto-generated by Godot. NEVER hand-edit. Add to .gitignore OR commit wholesale — do not selectively ignore.
- `project.godot` — Project config. Pin Godot version here.

## GDScript Stub Pattern

Syntactically valid GDScript. Every stub must follow this structure:

```gdscript
class_name CardData
extends Resource

@export var card_name: String = ""
@export var card_type: String = ""
@export var cost: int = 0

# TODO: core-systems agent implements — see FR-N in handoffs/brief.md
func resolve(context: Dictionary) -> Dictionary:
    return {}  # TODO
```

## Scene (.tscn) Stub Pattern

Minimal valid Godot 4 scene file. Godot rejects files with invalid headers.

```
[gd_scene format=3 uid="uid://placeholder"]

[node name="MainScene" type="Node2D"]
; TODO: ui-ux agent — see FR-N in handoffs/brief.md
```

**CRITICAL**: The header MUST be `[gd_scene format=3 uid="uid://..."]`. The `format=3` is required for Godot 4.x. `load_steps` is omitted when there are no external/sub resources.

## Resource (.tres) Stub Pattern

Minimal valid Godot 4 resource data file.

```
[gd_resource type="Resource" format=3]

[resource]
; TODO: content agent populates — requires Resource class defined first by core-systems
```

**CRITICAL**: The header MUST be `[gd_resource type="Resource" format=3]`. For custom typed resources with a script reference:

```
[gd_resource type="Resource" script_class="CardData" load_steps=2 format=3]

[ext_resource type="Script" path="res://scripts/resources/card_data.gd" id="1"]

[resource]
script = ExtResource("1")
```

## project.godot Minimal Config

Pin to current stable Godot 4.6:

```ini
; Godot Engine project configuration

[application]

config/name="ProjectName"
config/features=PackedStringArray("4.6")
run/main_scene="res://scenes/main.tscn"

[rendering]

renderer/rendering_method="mobile"
```

## Makefile Targets

```makefile
setup:      ## Install GUT plugin, install Python deps
	@echo "Installing GUT plugin..."
	mkdir -p addons/gut
	@echo "Installing Python simulation deps..."
	pip install -r simulation/requirements.txt

test-gd:    ## Run GUT tests headless
	godot -d -s --path "$$PWD" addons/gut/gut_cmdln.gd -gdir=res://tests/gut -ginclude_subdirs -gexit

test-py:    ## Run Python simulation tests
	pytest simulation/tests/

test: test-gd test-py  ## Run all tests

simulate:   ## Run Python headless simulation
	python simulation/runner.py

run:        ## Open in Godot editor
	godot --editor .
```

## Headless Simulation Pattern

- Python simulation lives in `simulation/` at project root — NOT inside `res://`
- It mirrors game balance logic in pure Python (no Godot dependency)
- This lets the simulation agent validate game formulas without launching the Godot engine
- Use `godot --headless` for integration tests that need the full engine

## Destructive Operations (require human gate)

- Deleting any `.tres` data file (data loss risk)
- Modifying `project.godot` (affects all agent version assumptions)
- Restructuring `res://` paths (breaks all `load()` and `preload()` calls across all .gd files)

## Narrative Agent Pattern (AI text generation in-game)

For games where agents call Claude API at runtime:
- Requires `ANTHROPIC_API_KEY` in `.env` — document in `.env.example` with expected call volume
- Cache generated text keyed by run_id to avoid redundant API calls
- Document estimated cost per session in `.env.example` comments
- Never hardcode API key in any script

## Testing Pattern

```
tests/gut/                  ← GDScript tests (GUT framework, v9.6.0), run headless
simulation/tests/           ← Python tests for simulation logic (pytest)
USER_TESTING.md             ← manual play scenarios, one per FR
```

GUT headless command: `godot -d -s --path "$PWD" addons/gut/gut_cmdln.gd -gdir=res://tests/gut -ginclude_subdirs -gexit`

Exit codes: 0 = all pass, 1 = any fail.
